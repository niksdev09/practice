# Практика по компьютерному зрению с YOLOv8. Тема 4.

Практическая работа, в ходе выполнения которой мы научимся выполнять детекцию объектов, используя модель YOLOv8. В данном репозитории указаны все этапы выполнения практики, от базовой детекции до высчитывания метрик.

## Архитектура модели

В проекте используется модель **YOLOv8** (You Only Look Once version 8) - современная нейронная сеть для детекции объектов в реальном времени. А в данной работе будет использоваться
- `yolov8n.pt` - nano версия (самая быстрая, но менее точная)

### Как устроена YOLOv8

Модель YOLOv8 работает в три этапа:

1. **Backbone** 
   - Берет картинку и находит в ней важные детали
   - Работает как "глаза" модели - смотрит на изображение и выделяет ключевые части

2. **Neck** 
   - Соединяет информацию с разных уровней
   - Помогает найти как большие, так и маленькие объекты на картинке

3. **Head** 
   - Делает финальные выводы - где объект и что это за объект
   - Рисует прямоугольники вокруг найденных объектов и отображает, насколько уверена в своем предсказании

## Датасет

### COCO Dataset
Для этапов 1-8 используется предобученная модель на датасете COCO, который содержит 80 классов объектов (люди, животные, фрукты, предметы. 

### Собственный датасет фруктов
В 8 этапе используется датасет с фруктами:
- **Классы**: apple (яблоко), banana (банан), orange (апельсин)
- **Формат**: изображения в формате JPG с XML-аннотациями в формате PASCAL VOC
- **Расположение**: [tasks/8/dataset/](./tasks/8/dataset)

## Описание этапов

- **[Этап 1](./tasks/1/)** - Первый запуск и использование YOLOv8 для детекции объектов на изображении
- **[Этап 2](./tasks/2/)** - Получение детальной информации о найденных объектах (координаты, уверенность, классы)
- **[Этап 3](./tasks/3/)** - Применение детекции к видеопотоку с сохранением результатов
- **[Этап 4](./tasks/4/)** - Детекция только определенных типов объектов (например, только людей)
- **[Этап 5](./tasks/5/)** - Реальное время: детекция с веб-камеры с подсчетом количества объектов каждого типа
- **[Этап 6](./tasks/6/)** - Область интереса ROI и как она влияет на детекцию только в заданной области кадра
- **[Этап 7](./tasks/7/)** - Влияния порога уверенности на количество детекций
- **[Этап 8](./tasks/8/)** - Оценка качества распознования, расчет метрик (Precision, Recall, F1) на реальном датасете фруктов

## Установка и запуск

### Требования
- Python 3.8+
> [!IMPORTANT]
>### Установка
> 
>Установите необходимые зависимости:
>```bash
>pip install ultralytics opencv-python
>```
>
>### Запуск
>
>Каждый этап запускается независимо. Перейдите в нужную папку и запустите скрипт:
>
>```bash
>cd tasks\1
>python main.py
>```

Для этапов с видео или веб-камерой:
- Для этапа 3 убедитесь, что файл `video.mp4` находится в папке
- В этапах 5 и 6 требуется подключенная веб-камера

### Также важно

- Результаты детекции на изображениях сохраняются с суффиксом `_result`
- Модели YOLOv8 автоматически скачиваются при первом запуске, если их нет в папке

### Как улучшить детектирование объектов
- Можно попробовать использовать более точную модель, например yolov8s.pt или yolov8m.pt, но опять же нужно понимать, что с повышение точности растет время детекции.
- Отрегулировать порог уверенности, параметр conf.
- Можно также обучить модель на собственном датесете, если перед нами стоит задача искать только определенные фрукты.
